## Context

参考基线的职责已经清楚：

- Firefly：借系统化博客结构与布局层级
- Coff0xc：借成熟内容组织和真实博客完成度
- Anna：借首页 Hero 的主星体气质与深空氛围

TZBlog 需要做的不是拼贴三者，而是：用 Firefly 的系统感、Coff0xc 的内容成熟度、Anna 的 Hero 气质，形成“深空观测站”自己的产品表达。

## Goals / Non-Goals

**Goals**

- 让首页更像真实博客产品，而不是概念展示页
- 把 Hero 从“主表演区”收束成“气质锚点 + 身份表达 + CTA”
- 提高内容模块的密度、节奏与运营感
- 提升导航、卡片和版心结构的成熟度
- 增加低噪声、可降级的动效反馈

**Non-Goals**

- 本轮不引入重型 3D 运行时或复杂 shader
- 本轮不做依赖 Docker/数据库的运行时验证
- 本轮不扩展新业务类型，仍围绕 posts/projects/docs/notes/pages

## Design Directions

### 1. 首页信息权重重排
- Hero 仍保留，但缩短纵向占比
- 最新内容 / 重点项目 / 文档更新显著前置
- 首页中段成为真正的内容分发中枢

### 2. Hero 气质重构
- 保留主星体与轨道感
- 减少“组件面板堆叠感”
- 前景内容更少、更硬、更利落
- 把宇宙感集中在 Hero，不向正文泛滥

### 3. 导航与卡片成熟化
- 顶部导航更短、更稳、更像长期站点
- 卡片统一信息优先级、标签层级、meta 节奏
- 列表页更接近真实博客运营中状态

### 4. 轻动效边界
- 页面切换：轻量
- 卡片 hover：微抬升 + 边框/光晕反馈
- 导航：激活态航迹线
- Hero：弱呼吸/弱视差/低频轨道反馈

## Validation Plan

- 只运行轻量验证：`astro check`、`astro build`
- 由用户在自己的机器上验证真实浏览效果并反馈
