## Context

TZBlog 前台界面已经从占位页升级为完整页面系统，但仍需要从“审计后修复”的角度做第二轮整理。这一轮应聚焦质量，而不是扩大功能面。

## Goals / Non-Goals

**Goals**

- 检查页面模板与布局系统的一致性
- 检查明显的可访问性、SEO、语义和响应式问题
- 识别未来接 Payload 前应先修的前台结构问题
- 在不偏离主设计方向的前提下修复问题

**Non-Goals**

- 本轮不接入真实 Payload 数据
- 本轮不扩展新页面类型
- 本轮不引入重型动画与 3D

## Audit Axes

1. 路由与导航完整性
2. 页面标题、描述与语义结构
3. 模板复用与代码组织
4. 响应式与布局稳定性
5. 搜索页面与交互壳层可用性
6. 未来接 CMS 时的数据契约可替换性

## Execution Rule

- 先列问题，再修复
- 每修一轮都复跑 `astro check` 与 `astro build`
- 只做和问题直接相关的改动，避免边审边扩需求
