## Why

当前 TZBlog 前台已经具备完整页面骨架，但整体观感仍偏“设计稿原型 / 示例界面”，与 Firefly、Coff0xc、Anna 这类成熟站点相比，主要差距在于：首页信息权重、导航成熟度、内容区完成度、Hero 气质整合和动效克制表达。

用户已经明确希望 TZBlog 具备可与这些参考站比肩的效果，而不是停留在“有深色主题和行星就算完成”的阶段。

## What Changes

- 重构首页信息权重，让内容分发层明显强于纯 Hero 展示感
- 收敛 Hero 的视觉结构，强化“Anna 式主星体气质 + TZBlog 观测站表达”
- 提升导航、卡片、模块、列表页的成熟度，使前台更像真实运营博客而不是示例面板
- 为轻量动效建立更明确的层级：页面反馈、卡片反馈、航迹感、首屏弱动态
- 为后续接 Payload 真数据保留稳定的布局与内容契约

## Capabilities

### Modified Capabilities

- `platform-foundation`: 将前台从“完整骨架”提升为“更接近成熟博客产品的内容优先观测站界面系统”

## Impact

- 主要影响 `apps/web/src/**`
- 可能同步更新设计/流程文档中的前台完成度描述
- 不改动锁定技术栈，不引入重型运行时依赖
