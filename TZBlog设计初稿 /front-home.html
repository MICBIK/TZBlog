spec-first 的核心：先写「验收脚本」，再让 agent 去满足它

下面是我每个功能开工前都会先落的最小验证层——它比任何 prompt 技巧都更能让 agent 一次写对：

bashverify.sh
1234567
# 验收脚本先于实现存在——agent 的唯一目标是让它全绿
set -euo pipefail

pnpm tsc --noEmit              # 类型先过
pnpm vitest run spec/checkout  # 行为契约
node scripts/replay.js "last-failure"  # 回放上次失败轨迹
echo "✓ spec satisfied"

关键不是脚本写得多漂亮，而是它在 agent 动手之前就存在——这把"我以为它懂了"变成"它必须通过的闸门"。
