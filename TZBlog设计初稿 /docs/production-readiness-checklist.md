# tzblog 上线就绪缺口清单

> 评估对象：当前原型（16 个静态 HTML 页面，终端暗色主题，`site-chrome` 前台共享层 + `admin-chrome` 后台共享层）。
> 评估口径：原型已完成「界面设计 + 客户端交互演示」，但所有数据为硬编码 mock、无运行后端。本清单列出从「可演示原型」到「可上线运行」之间的全部缺口。
> 优先级：**P0 = 上线必须**（缺了不能上）／**P1 = 重要**（上线前应补，影响质量/体验/SEO）／**P2 = 增强**（上线后迭代）。
> 状态约定：`[现状]` = 原型里已有的；`[缺口]` = 需补的。

---

## A. 后端 · 数据 · 鉴权 — P0

- [ ] **真实后端服务**：`[现状]` 无后端，数据写死在每页内联 `<script>` 常量。`[缺口]` 按手册 §6 建议契约实现 Go API + Postgres，统一响应信封 `{success,data,error,meta}`。
- [ ] **数据库与建表**：`[缺口]` 落地手册 §7 数据模型（Post/User/Comment/Category/Pathway/Media/Settings），含索引、外键、软删除、时间戳。
- [ ] **真实鉴权**：`[现状]` `auth.html` 是静态表单。`[缺口]` GitHub/Google OAuth 回调、邮箱+密码、magic-link 真实流程；session/JWT 签发与校验；登录态在前台/后台的真实门槛。
- [ ] **权限分级落地**：`[缺口]` 匿名可读、注册解锁点赞/收藏/下载/跨端同步、管理员后台准入的服务端校验（不能只靠前端隐藏）。
- [ ] **数据注入点替换**：`[现状]` mock 常量。`[缺口]` 按手册 §9 把每页常量数组切成真实 `fetch`，补加载态/错误态/空态。

## B. 内容渲染管线 — P0

- [ ] **Markdown/MDX 管线**：`[现状]` 编辑器是轻量正则 MD 解析（仅演示）。`[缺口]` 服务端/构建期真实 MD→HTML，支持 frontmatter、目录生成、脚注、表格、任务列表。
- [ ] **代码高亮 Shiki**：`[现状]` 文章页代码块是静态高亮样式。`[缺口]` 接入 Shiki 真实多语言高亮 + 行高亮/行号。
- [ ] **公式 KaTeX**：`[缺口]` 接入 KaTeX 渲染行内/块级公式。
- [ ] **XSS 净化**：`[缺口]` 评论与任何用户产出的 Markdown 必须经 sanitize（白名单）后渲染——当前无防护。

## C. 缺失页面与状态 — P0 / P1

- [ ] **独立错误页 404 / 500** — P0：`[现状]` 仅有 `site-chrome` 的运行时错误 toast。`[缺口]` 终端风格的独立 404/500 页面（保持主题）。
- [ ] **空状态设计** — P1：`[现状]` 搜索无结果有引导文案。`[缺口]` 无文章/无评论/无媒体/无草稿等空态统一设计。
- [ ] **加载骨架逐页应用** — P1：`[现状]` `site-chrome` 提供 `.skeleton` 工具类但未逐页落地。`[缺口]` 列表/文章/看板首屏接入骨架屏。
- [ ] **文章列表分页 / 无限滚动** — P1：`[缺口]` 首页/归档/搜索结果的分页或滚动加载（契约 §6.3 已留 `meta` 分页位）。
- [ ] **标签页 / 分类归档页** — P1：`[现状]` 有热门标签 chip 与五条内容线。`[缺口]` `tag/:slug`、`category/:slug` 独立归档页。
- [ ] **RSS/Atom、sitemap.xml、robots.txt** — P0（内容站标配）：`[缺口]` 全部缺失，SEO/订阅依赖它们。
- [ ] **评论闭环** — P1：`[现状]` 文章页可输入+静态展示。`[缺口]` 真实发布、嵌套回复、审核队列联动（后台 dashboard 已有审核 UI 占位）、防刷。

## D. 用户态闭环 — P1

- [ ] **登录后个人中心**：`[缺口]` 我的收藏/点赞/阅读历史/资料设置/跨端同步页面。
- [ ] **点赞/收藏/阅读进度持久化**：`[现状]` 文章页为客户端切换演示。`[缺口]` 落库 + 登录态关联（契约 §6.8）。
- [ ] **订阅/Newsletter**：`[现状]` landing 有邮箱输入 + toast。`[缺口]` 真实订阅入库与发信。

## E. 可访问性 (a11y) — P1

- [ ] **对比度核对**：`[缺口]` 磷光绿 `#3fe08f` 于深底、`--muted` 文字需过 WCAG AA（4.5:1）核对，暗底低饱和文字易不达标。
- [ ] **键盘导航 + focus-visible**：`[缺口]` 全站可聚焦元素的可见焦点环、Tab 顺序、跳过导航 skip-link。
- [ ] **语义与 ARIA**：`[现状]` 搜索 chip 有 `aria-pressed`。`[缺口]` 全站 landmark/标题层级/表单 label/图标按钮 aria-label 审计。
- [ ] **自定义光标的可达性**：`[缺口]` 确认自定义光标不影响放大镜/高对比模式用户；`prefers-reduced-motion` 已处理动效。

## F. 性能 · SEO — P1

- [ ] **公共资源抽取**：`[现状]` 每页 CSS/JS 内联（为隔离主题）。`[缺口]` 生产需抽公共 bundle + 缓存，否则重复体积大、无法跨页缓存。
- [ ] **SEO 元信息**：`[缺口]` 每页 `<title>`/`description`/canonical/OpenGraph/Twitter Card/JSON-LD（Article/BreadcrumbList）结构化数据。
- [ ] **图片管线**：`[现状]` 占位块，无真实图。`[缺口]` 真实图 + 尺寸标注 + 懒加载 + AVIF/WebP + 封面/OG 图生成。
- [ ] **字体策略**：`[缺口]` webfont 子集化 + `font-display:swap`，避免 FOUT/首屏抖动。
- [ ] **Core Web Vitals**：`[缺口]` LCP/INP/CLS 实测达标（手册性能口径）。

## G. 响应式 · 跨设备 — P1

- [ ] **移动端逐页核对**：`[缺口]` 顶栏抽屉、宽表格在窄屏、编辑器三栏在窄屏、看板图表在窄屏的真实可用性（360/390/430/768 无横向滚动）。
- [ ] **后台移动端**：`[现状]` `admin-chrome` 侧栏移动端抽屉。`[缺口]` 后台表格/编辑器在平板/手机的可操作性核对。

## H. 安全 · 合规 — P0

- [ ] **CSP 与安全响应头**：`[缺口]` 生产 CSP（nonce 脚本）、HSTS、X-Content-Type-Options、X-Frame-Options、Referrer-Policy。
- [ ] **CSRF / 限流**：`[缺口]` 状态变更接口 CSRF 防护、登录/评论/订阅端点限流与反滥用。
- [ ] **ICP 备案号**：`[现状]` 底栏已留备案号占位。`[缺口]` 填真实备案号（中国大陆主体 `tzcode.top` 必须）。
- [ ] **密钥与配置**：`[缺口]` OAuth secret / DB 凭据走环境变量与密钥管理，不入库不入前端。

## I. 工程化 · 交付 — P1 / P2

- [ ] **迁移到 Next.js** — P1：`[缺口]` 把自包含 HTML 拆为组件 + 路由 + SSR/SSG；token 落地为 CSS 变量/Tailwind config；`site-chrome`/`admin-chrome` 对应根布局与后台布局组件（手册 §2.1）。
- [ ] **组件库抽取** — P1：`[缺口]` 按钮/卡片/表单/表格/侧栏/底栏/toast 等沉淀为复用组件（底栏与后台侧栏已是数据驱动单一数据源，可直接对应）。
- [ ] **测试** — P1：`[缺口]` 单元（工具/数据转换）、集成（API）、E2E（关键流：搜索/阅读/登录/发文）、视觉回归（主题一致性）。
- [ ] **CI/CD + 监控** — P2：`[缺口]` 构建/部署流水线、错误监控（Sentry 类）、访问统计真实接入（后台 analytics 当前为 mock）。

## J. 体验增强 — P2

- [ ] **亮色主题**：`[缺口]` 当前仅暗色终端风；如需日间模式需设计亮色 token 与切换。
- [ ] **国际化**：`[缺口]` 当前中文为主；如面向海外华人/SEO 需 i18n 方案。
- [ ] **站内通知 / 站内搜索增强 / 相关文章推荐**：`[缺口]` 视运营需要迭代。

---

## 一句话结论

原型已经把**界面、视觉系统、客户端交互演示、共享层架构**做扎实了；距离上线，核心缺口集中在 **A 后端数据鉴权 / B 内容渲染管线 / H 安全合规 / C 错误页·RSS·sitemap** 这四块 **P0**——它们决定「能不能上」；P1 决定「上得好不好」；P2 是上线后迭代。建议按 A→B→H→C 的顺序拉起最小可上线闭环。
