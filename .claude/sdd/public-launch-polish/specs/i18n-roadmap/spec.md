# i18n-roadmap spec

## SCENARIO: i18n-roadmap-001

**GIVEN** the project advertises `SUPPORTED_LOCALES = ["zh", "en"]`
**WHEN** launch readiness debt is reviewed
**THEN** the V3 backlog records that the current implementation is single-locale at route, metadata, RSS, sitemap, static UI, and service-helper levels

## SCENARIO: i18n-roadmap-002

**GIVEN** future work starts on full i18n
**WHEN** engineers read the project context
**THEN** they see a concrete implementation direction aligned with Next.js App Router locale routing and dictionary loading
