// 认证中间件

### 2. D2c: 路由参数冲突

**问题描述**:
```
panic: 'by-id/:id/comments' in path '/api/v1/articles/by-id/:id/comments' conflicts with existing wildcard ':slug' in existing prefix '/api/v1/articles/:slug'
```

**根本原因**:
- 文章路由同时使用 `:slug` 和 `:id` 在同一层级
- Gin 不允许同一层级混用不同参数名

**修复内容**:

**路由调整**:
```go
// 评论路由 - 使用 by-id 前缀
articles.GET("/by-id/:id/comments", commentHandler.ListComments)

// 文章 CRUD - 统一使用 slug
articles.PUT("/:slug", articleHandler.UpdateArticle)
articles.DELETE("/:slug", articleHandler.DeleteArticle)
```

**Handler 调整**:
```go
// UpdateArticle - 先通过 slug 查询文章
func (h *ArticleHandler) UpdateArticle(c *gin.Context) {
    slug := c.Param("slug")
    
    // 1. 先查询文章获取 ID
    article, err := h.service.GetArticleBySlug(c.Request.Context(), slug)
    if err != nil {
        // ...
        return
    }
    
    // 2. 使用查到的 ID 进行更新
    err = h.service.UpdateArticle(c.Request.Context(), article.ID, dto)
    // ...
}
```

**验证结果**:
- ✅ 服务器正常启动（不再 panic）
- ✅ 所有路由无冲突
- ✅ SEO 友好（使用 slug）
- ✅ 评论路由正常工作
